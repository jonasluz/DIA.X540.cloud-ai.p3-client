# Project #3: Chalice Application using Artificial Intelligence AWS Services - Client Application

## Description

This is the source of the client **Unity** application for the submission of the Project #3 of the X-540 course.

This code does represent only the client side of the submission and this repository is used as a aubmodule of the [main repository](https://github.com/jonasluz/DIA.X540.cloud-ai), that keeps the full source code (by adding this as a submodule) and the [full Report of the submission](https://github.com/jonasluz/DIA.X540.cloud-ai/blob/main/Project-3/docs/REPORT.md). Please, refer to the main repository for this submission.

Use the release of this repository to download and test the client application, which connects to the backend services deployed on AWS. This system is garanteed to work, at least, until 2025 December 31. 

The following section explain the installation procedure for this application. Please, take into account that the application is designed to work in Brazilian Portuguese.

## Installing and running the client aplication

1. Open the latest release page.
2. Download the version for your operating system. 
3. Unpack the downloaded file.

To run the application, follow the instruction for your operating system: 

### For Windows

1. Run the executable file: `DIA.X540.cloud-ai.p3-client.exe`
2. The system will emit a warning that the program wants to use the microphone. It is required for the application to work. Allow it.

### For Linux

1. Run the executable file: `PPGIA-X540-P3-MVP-3.20251130.x86_64`
2. The system will emit a warning that the program wants to use the microphone. It is required for the application to work. Allow it.

### For Mac

1. Run the unpacked application: `PPGIA-X540-P3-MVP-3.20251130-mac.app`
2. The system will avoid running the application because it is not signed.
3. Choose to open the application content in Finder.
4. Click with your mouse secondary button on the application icon and select **Run**
5. The system will again alert you about the application not being signed, but will allow you to run it anyways.
6. Select to run the unsigned application. This procedure will not be necessary for this version of the application anymore.
7. The system will emit a warning that the program wants to use the microphone. It is required for the application to work. Allow it.

### For Linux

1. Install the apk file in your device.
2. The system will forbid installation because the apk was downloaded from an insecure source.
3. Accept to open configuration and allow the installation from this source.
4. Try yo install the apk again. 
5. After installing, run the installed application, by touching its icon.
6. The system will emit a warning that the program wants to use the microphone. It is required for the application to work. Allow it.
